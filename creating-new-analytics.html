Creating New Analytics

Taking a step back for a second - how do we define an analytic? What is it made up of? What does it do? How does it work?

An analytic is a collection of functions. There are inputs, outputs, and all the stuff that goes on in-between. Functions help to reduce code duplication and introduce standardization since there is now one way of performing one task.

The rdww.yaml package provides a handy feature for creating new analytics. This is especially helpful if you:

  • find yourself doing the same task over and over,

  • have been carrying a stray .R file with you for the last couple years to copy-paste across projects, or

  • just want the freedom of your collection of functions not being as “hands-on” every time you run it.

create_from_template()
'
$name
[1] "Name of Analytic"

$description
[1] "Short, descriptive summary of analytic."

$script
[1] "path/to/script.ext"

$datasets

$tableau_shell

[1] "[Client Name] - [Analytic Name] - [Plan Name] - FY[XX]"

$memos

[4] "Summary Memo"
'

create_from_template(
  name = "Cool New Analytic",
  description = "This analytic does some really cool things!",
  script = "path/to/analytic/script.R",
  datasets = c("First Dataset", "Dataset 2", "Another Dataset")
)
'

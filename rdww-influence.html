rdww-influence

The rdwwverse package is an attempt at reducing the effort required to complete an analytic’s lifecycle. The methods implemented throughout this network of packages accomplish this goal by abstracting commonly used functions to their highest-level, favoring fewer, generic functions over plentiful, specific functions. This abstraction - we believe - should make even the least familiar R programmer comfortable completing an analytic from start to finish.

Design Principles

Although not perfect, this ecosystem prioritizes human centered development. Every function shares this common goal by asking the following questions:

  • Is each function and its parameters easy to remember?

  • Does each function call read like English?

  • Does each function do what the user expects?

Of course, there are assumptions we make in answering each question, assumptions we continuously try to validate by talking with current/potential users of the package. Facilitating these conversations and making appropriate changes are extremely simple when the user is prioritized from the beginning. We will always support conversations whose intentions concern the end user.

Technically speaking, we also hope to prove there is more in doing less. The source code is minimal and takes advantage of concepts such as functional programming and vectorization to reduce the need for repetition. This flexible approach has allowed for rapid development of this ecosystem. Paired with the desire to form a community around this package, the time required between question and answer should be negligible.
